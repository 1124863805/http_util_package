# 万年历组件：样式限制与交互体验分析

## 一、样式不足（限制）

### 1. 休/班角标
- **重叠风险**：预留区域（top/right 14px）在小格子或大字号下仍可能与日期数字边缘相切；FittedBox 缩放后中心偏移，角标与数字仍可能挨得太近。
- **位置单一**：休、班均在右上角，节假密集时整列角标视觉重复；部分产品将「休」放左上、「班」放右上以区分，当前未区分。
- **与边框关系**：有「今天」描边时，角标紧贴边框（top: 4, right: 6），略显挤。

### 2. 格子内层次
- **公历与农历**：仅靠字号和字重区分，农历/节气/节日在选中或今天时统一为主色，层次感可再加强（如节气保持 tertiary、节日保持 error 的色相）。
- **其他月补全**：仅透明度 0.5，与当月对比可再明显一点（如更小字号或更浅色），避免「抢戏」。

### 3. 星期头与格子对齐
- **列宽**：星期头用 `Row` + `Expanded`，下方格子用 `GridView`，理论上 7 列对齐；若 padding 不一致或 Grid 的 crossAxisSpacing 导致列宽与头不对齐，会显得错位。
- **周末列**：已取消周末列背景后，仅靠「日」「六」红色区分，整列无背景，弱设备上可能不够明显。

### 4. 字体与整体气质
- 已用霞鹜文楷，若未正确加载会回退到系统字体，视觉上像「普通 App」而非传统历。
- 农历/节气/节日未再做字重或字号的细分，传统历的「节气略大、农历略小」的层级可再细化。

### 5. 色彩体系
- 节日红、节气 tertiary、主色描边均依赖 Theme；在浅色/深色主题下若 contrast 不足，可读性会下降。
- 无「宜/忌」等黄历扩展，属功能范围，但若未来要加，当前格子信息密度已较高，需预留空间或折叠。

## 二、交互体验

### 1. 滑动
- **左右滑月**：PageView 默认有滑动，但无「上一月/下一月」的明确提示（如两侧浅箭头或首尾月提示），新用户可能不知道可滑。
- **惯性/边界**：滑到 1970 或 2070 附近无阻尼或提示，容易一直滑到底才意识到到顶。
- **与星期头的关系**：星期头固定、仅月历滑动，符合预期；无联动动画（如月份标题随滑轻微位移）属于可接受取舍。

### 2. 点击
- **命中区域**：整个格子可点，无 min touch target 约束，大屏上没问题；小屏格子小时可能略难点，可考虑略微扩大点击热区（如 padding 或 InkWell 的 splash）。
- **反馈**：仅有 setState 重绘选中态，无 ripple 或短震动，反馈略弱。
- **选中与「今天」**：选中与今天都用主色描边，今天若未选中则只有描边无填充，逻辑清晰；若产品希望「今天」永远高亮且可选，当前实现已满足。

### 3. 信息获取
- **无长按/二级信息**：点击即选中，无长按显示详情（如完整农历、节气时刻、宜忌等），信息深度有限。
- **无月份快跳**：只能逐月滑动，无法输入年月或选择月份快速跳转，在「看很久以后/以前」时效率低。

### 4. 可访问性
- **无语义/无障碍**：未对「今天」「选中」「休」「班」等做 Semantics，读屏与自动化测试支持不足。
- **对比度**：依赖系统主题，未单独检查 4.5:1 等对比度，在部分主题下可能不达标。

## 三、可改进优先级建议

| 优先级 | 项           | 说明 |
|--------|--------------|------|
| 高     | 休/班不重叠  | 加大预留区或角标外置；必要时休左上、班右上分离。 |
| 高     | 点击反馈     | 增加 InkWell/水波纹或轻触反馈。 |
| 中     | 星期头与格对齐 | 统一 padding 与 Grid 的 spacing，保证 7 列严格对齐。 |
| 中     | 滑动边界提示 | 接近首/末月时弱提示或阻尼。 |
| 低     | 月份快跳     | 提供月份选择器或输入年月。 |
| 低     | 语义/无障碍  | 为今天、选中、休、班补充 Semantics。 |
